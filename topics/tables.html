HTML Tables

What are Tables?

HTML tables are used to display data in rows and columns. They're perfect for presenting structured information like spreadsheets, price lists, schedules, or comparison charts.

Basic Table Structure

Every table is built with these essential elements:

  • <table> - Container for the entire table
  • <tr> - Table row
  • <td> - Table data cell
  • <th> - Table header cell

Example: Simple Table

<table>
    <tr>
        <th>Name</th>
        <th>Age</th>
        <th>City</th>
    </tr>
    <tr>
        <td>John</td>
        <td>25</td>
        <td>New York</td>
    </tr>
    <tr>
        <td>Jane</td>
        <td>30</td>
        <td>London</td>
    </tr>
</table>

Result:

Name Age City
John 25 New York
Jane 30 London

Table Sections

For better structure and accessibility, organize tables into sections:

  • <thead> - Table header section
  • <tbody> - Table body section
  • <tfoot> - Table footer section

Example: Table with Sections

<table>
    <thead>
        <tr>
            <th>Product</th>
            <th>Price</th>
            <th>Quantity</th>
            <th>Total</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Laptop</td>
            <td>$999</td>
            <td>2</td>
            <td>$1,998</td>
        </tr>
        <tr>
            <td>Mouse</td>
            <td>$25</td>
            <td>3</td>
            <td>$75</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <td colspan="3"><strong>Grand Total</strong></td>
            <td><strong>$2,073</strong></td>
        </tr>
    </tfoot>
</table>

Result:

Product Price Quantity Total
Laptop $999 2 $1,998
Mouse $25 3 $75
Grand Total $2,073

Spanning Cells

Make cells span multiple columns or rows using colspan and rowspan:

Column Spanning (colspan)

Example: Colspan

<table>
    <tr>
        <th colspan="3">Student Grades</th>
    </tr>
    <tr>
        <th>Name</th>
        <th>Math</th>
        <th>Science</th>
    </tr>
    <tr>
        <td>Alice</td>
        <td>95</td>
        <td>88</td>
    </tr>
    <tr>
        <td>Bob</td>
        <td>87</td>
        <td>92</td>
    </tr>
</table>

Result:

Student Grades
Name Math Science
Alice 95 88
Bob 87 92

Row Spanning (rowspan)

Example: Rowspan

<table>
    <tr>
        <th>Department</th>
        <th>Employee</th>
        <th>Position</th>
    </tr>
    <tr>
        <td rowspan="2">IT</td>
        <td>John Smith</td>
        <td>Developer</td>
    </tr>
    <tr>
        <td>Sarah Jones</td>
        <td>Designer</td>
    </tr>
    <tr>
        <td rowspan="2">Marketing</td>
        <td>Mike Wilson</td>
        <td>Manager</td>
    </tr>
    <tr>
        <td>Lisa Brown</td>
        <td>Coordinator</td>
    </tr>
</table>

Result:

Department Employee Position
IT John Smith Developer
Sarah Jones Designer
Marketing Mike Wilson Manager
Lisa Brown Coordinator

Table Accessibility

Make tables accessible to screen readers and assistive technologies:

Example: Accessible Table

<table>
    <caption>Course Schedule for Computer Science Students</caption>
    <thead>
        <tr>
            <th scope="col">Time</th>
            <th scope="col">Monday</th>
            <th scope="col">Tuesday</th>
            <th scope="col">Wednesday</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <th scope="row">9:00 AM</th>
            <td>Programming</td>
            <td>Database</td>
            <td>Web Design</td>
        </tr>
        <tr>
            <th scope="row">11:00 AM</th>
            <td>Math</td>
            <td>Algorithms</td>
            <td>Networks</td>
        </tr>
    </tbody>
</table>

Result:

Course Schedule for Computer Science Students
Time Monday Tuesday Wednesday
9:00 AM Programming Database Web Design
11:00 AM Math Algorithms Networks

Accessibility Attributes

  • scope="col" - Indicates header applies to column
  • scope="row" - Indicates header applies to row
  • headers - Links cells to their headers (for complex tables)
  • caption - Provides table description
  • summary - Describes table structure (deprecated, use caption instead)

Table Best Practices

  • Use tables for tabular data only - Don't use tables for layout
  • Always include headers - Use <th> elements with appropriate scope attributes
  • Add captions - Provide context with <caption> elements
  • Structure properly - Use <thead>, <tbody>, and <tfoot> when appropriate
  • Keep it simple - Avoid overly complex table structures
  • Make it responsive - Consider mobile users with CSS
  • Test accessibility - Ensure screen readers can navigate your tables
  • Style consistently - Use CSS for visual formatting, not HTML attributes

When NOT to Use Tables

❌ Don't Use Tables For:

  • Page layout and positioning
  • Creating columns of text
  • Aligning form elements
  • Navigation menus
  • Any non-tabular content

✅ Do Use Tables For:

  • Displaying spreadsheet-like data
  • Comparison charts
  • Financial reports
  • Schedules and timetables
  • Any data with rows and columns relationship

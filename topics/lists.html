HTML Lists

Unordered Lists

Use <ul> for bullet-point lists where order doesn't matter:

Example: Unordered List

<ul>
    <li>Apple</li>
    <li>Banana</li>
    <li>Orange</li>
</ul>

Result:

  • Apple
  • Banana
  • Orange

Ordered Lists

Use <ol> for numbered lists where order matters:

Example: Ordered List

<ol>
    <li>First step</li>
    <li>Second step</li>
    <li>Third step</li>
</ol>

Result:

  1. First step
  2. Second step
  3. Third step

Ordered List Attributes

You can customize ordered lists with attributes:

Example: List Attributes

<!-- Start at 5 -->
<ol start="5">
    <li>Item 5</li>
    <li>Item 6</li>
</ol>

<!-- Use letters -->
<ol type="a">
    <li>Item a</li>
    <li>Item b</li>
</ol>

<!-- Use Roman numerals -->
<ol type="I">
    <li>Item I</li>
    <li>Item II</li>
</ol>

Description Lists

Use <dl> for term-definition pairs:

  • <dl> - Description list
  • <dt> - Description term
  • <dd> - Description definition

Example: Description List

<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language</dd>

    <dt>CSS</dt>
    <dd>Cascading Style Sheets</dd>

    <dt>JavaScript</dt>
    <dd>Programming language for web browsers</dd>
</dl>

Result:

HTML
HyperText Markup Language
CSS
Cascading Style Sheets
JavaScript
Programming language for web browsers

Nested Lists

Lists can be nested inside other lists:

Example: Nested Lists

<ul>
    <li>Fruits
        <ul>
            <li>Apple</li>
            <li>Banana</li>
        </ul>
    </li>
    <li>Vegetables
        <ul>
            <li>Carrot</li>
            <li>Broccoli</li>
        </ul>
    </li>
</ul>

Result:

  • Fruits
    • Apple
    • Banana
  • Vegetables
    • Carrot
    • Broccoli

List Best Practices

  • Use unordered lists for items where order doesn't matter
  • Use ordered lists for steps, rankings, or sequences
  • Use description lists for glossaries or term definitions
  • Keep list items concise and parallel in structure
  • Don't nest lists too deeply (3 levels maximum recommended)

HTML Key Terms and Glossary

Core Concepts

HTML (HyperText Markup Language)
The standard markup language used to create web pages. It describes the structure and content of a document using elements and tags.
Markup Language
A system for annotating text to indicate its structure, formatting, or presentation. HTML uses tags to mark up content.
HyperText
Text that contains links to other text or documents. The "hyper" part refers to the non-linear way of accessing information.
Element
A complete structure consisting of an opening tag, content, and closing tag. For example: <p>Hello</p>
Tag
Keywords enclosed in angle brackets that define HTML elements. Tags usually come in pairs: opening <tag> and closing </tag>

Example: Element vs Tag

<!-- This entire line is an ELEMENT -->
<p>This is a paragraph.</p>

<!-- These are TAGS -->
<p>      <-- Opening tag
</p>     <-- Closing tag

Result:

This is a paragraph.

Document Structure

DOCTYPE
Declaration that tells the browser which version of HTML the document uses. HTML5 uses <!DOCTYPE html>
Root Element
The <html> element that contains all other elements in the document.
Head Section
The <head> element containing metadata about the document (not visible to users).
Body Section
The <body> element containing all visible content of the webpage.
Metadata
Information about the document that doesn't appear on the page but helps browsers and search engines understand the content.

Example: Document Structure Terms

<!DOCTYPE html>          <-- DOCTYPE declaration
<html lang="en">           <-- Root element with attribute
<head>                     <-- Head section (metadata)
    <meta charset="UTF-8">
    <title>Page Title</title>
</head>
<body>                     <-- Body section (visible content)
    <h1>Welcome</h1>
</body>
</html>

Attributes and Values

Attribute
Additional information provided to an element, written inside the opening tag. Attributes modify or configure elements.
Attribute Value
The data assigned to an attribute, typically enclosed in quotes.
Global Attributes
Attributes that can be used on any HTML element (like id, class, style).
Boolean Attributes
Attributes that don't need values; their presence indicates true (like checked, disabled).

Example: Attributes and Values

<img src="photo.jpg"           <-- src is attribute, "photo.jpg" is value
     alt="Beautiful sunset"    <-- alt is attribute, "Beautiful sunset" is value
     width="300"               <-- width is attribute, "300" is value
     loading="lazy">           <-- loading is attribute, "lazy" is value

<input type="checkbox" checked>  <-- checked is boolean attribute

Content Types

Block-level Elements
Elements that take up the full width available and start on a new line (like <div>, <p>, <h1>).
Inline Elements
Elements that only take up as much width as needed and don't start on a new line (like <span>, <a>, <strong>).
Void Elements
Elements that don't have closing tags and don't contain content (like <img>, <br>, <hr>).
Semantic Elements
Elements that have meaning beyond their appearance (like <header>, <nav>, <article>).

Example: Element Types

<!-- Block-level elements -->
<div>This div takes full width</div>
<p>This paragraph also takes full width</p>

<!-- Inline elements -->
<span>This span</span> <strong>and this strong</strong> are inline.

<!-- Void elements -->
<img src="image.jpg" alt="Image">
<br>
<hr>

Result:

This div takes full width

This paragraph also takes full width

This span and this strong are inline.

Web Technologies

CSS (Cascading Style Sheets)
Language used to style HTML elements, controlling appearance, layout, and design.
JavaScript
Programming language that adds interactivity and dynamic behavior to web pages.
DOM (Document Object Model)
Programming interface that represents the HTML document as a tree structure that can be manipulated by JavaScript.
Browser
Software application that interprets and displays HTML documents (Chrome, Firefox, Safari, Edge).
Rendering
The process by which a browser converts HTML code into a visual webpage.

Accessibility Terms

Alt Text
Alternative text for images that screen readers can announce to visually impaired users.
ARIA (Accessible Rich Internet Applications)
Set of attributes that can be added to HTML elements to improve accessibility for assistive technologies.
Screen Reader
Assistive technology that reads web page content aloud for users who cannot see the screen.
Semantic HTML
Using HTML elements according to their intended meaning rather than just their appearance.

Example: Accessibility Features

<img src="chart.png" alt="Sales increased 25% from Q1 to Q2">

<button aria-label="Close dialog">×</button>

<nav aria-label="Main navigation">
    <ul>
        <li><a href="#home">Home</a></li>
        <li><a href="#about">About</a></li>
    </ul>
</nav>

Development Terms

Validation
The process of checking HTML code for errors and compliance with standards.
Nesting
Placing HTML elements inside other elements in a hierarchical structure.
Indentation
Adding spaces or tabs to make code structure more readable by showing element hierarchy.
Comments
Notes in the code that aren't displayed on the webpage, used to explain or organize code.
Responsive Design
Designing web pages that adapt to different screen sizes and devices.

Example: Good Code Practices

<!-- This is a comment explaining the navigation -->
<nav>
    <ul>                    <-- Proper nesting: ul inside nav
        <li>               <-- Proper indentation shows hierarchy
            <a href="#">Home</a>
        </li>
        <li>
            <a href="#">About</a>
        </li>
    </ul>
</nav>

Quick Reference

Most Important Terms for Beginners

  • Element: Complete HTML structure with tags and content
  • Tag: Keywords in angle brackets
  • Attribute: Additional information for elements
  • Semantic: Using elements for their meaning
  • Validation: Checking code for errors
  • Accessibility: Making content usable by everyone

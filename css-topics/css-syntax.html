CSS Syntax & Selectors

CSS Syntax

CSS syntax consists of rules that tell the browser how to style HTML elements. Each rule has two main parts: a selector and a declaration block.

Basic CSS Syntax Structure:

selector {
    property: value;
    property: value;
}

Anatomy of a CSS Rule:

h1 {                    /* ← Selector */
    color: #0C2292;     /* ← Property: Value; */
    font-size: 2rem;    /* ← Property: Value; */
}                       /* ← Declaration Block */

CSS Rule Components

  • Selector: Targets which HTML elements to style
  • Property: The aspect you want to change (color, size, etc.)
  • Value: How you want to change it
  • Declaration: A property-value pair
  • Declaration Block: All declarations wrapped in curly braces { }

✅ Correct Syntax:

p {
    color: blue;
    font-size: 16px;
    margin: 10px;
}

❌ Common Syntax Errors:

/* Missing semicolon */
p {
    color: blue
    font-size: 16px;
}

/* Missing closing brace */
p {
    color: blue;
    font-size: 16px;

/* Missing property name */
p {
    blue;
    font-size: 16px;
}

Types of CSS Selectors

1. Element Selectors

Select all elements of a specific HTML tag:

Example: Element Selectors

/* Selects ALL h1 elements */
h1 {
    color: #0C2292;
    font-size: 2rem;
}

/* Selects ALL paragraph elements */
p {
    color: #333;
    line-height: 1.6;
}

/* Selects ALL div elements */
div {
    background-color: #f8f9fa;
    padding: 15px;
}

Result:

This is an h2 element

This is a paragraph element.

This is a div element.

2. Class Selectors

Select elements with a specific class attribute using a dot (.):

Example: Class Selectors

/* HTML */
<p class="highlight">This paragraph has a class</p>
<p>This paragraph has no class</p>
<div class="highlight">This div also has the same class</div>

/* CSS */
.highlight {
    background-color: #E8BB17;
    color: white;
    padding: 10px;
    border-radius: 5px;
}

Result:

This paragraph has the "highlight" class

This paragraph has no class

This div also has the "highlight" class

3. ID Selectors

Select a unique element with a specific ID using a hash (#):

Example: ID Selectors

/* HTML */
<p id="special">This paragraph has a unique ID</p>
<p>This paragraph has no ID</p>

/* CSS */
#special {
    background-color: rgba(12, 34, 146, 0.2);
    color: #0C2292;
    border: 2px solid #0C2292;
    padding: 15px;
    font-weight: bold;
}

Result:

This paragraph has the "special" ID

This paragraph has no ID

⚠️ Important Note:

IDs must be unique - only one element per page can have the same ID!

4. Universal Selector

Select ALL elements using an asterisk (*):

Example: Universal Selector

/* Applies to ALL elements */
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

Common Use: CSS resets to remove default browser styling.

5. Attribute Selectors

Select elements based on their attributes:

Example: Attribute Selectors

/* Select inputs with type="text" */
input[type="text"] {
    border: 2px solid #0C2292;
    padding: 8px;
}

/* Select links with target="_blank" */
a[target="_blank"] {
    color: #E8BB17;
    font-weight: bold;
}

/* Select elements with a specific class */
[class="highlight"] {
    background-color: yellow;
}

Attribute Selector Variations:

  • [attribute] - Has the attribute
  • [attribute="value"] - Exact value match
  • [attribute^="value"] - Starts with value
  • [attribute$="value"] - Ends with value
  • [attribute*="value"] - Contains value

Pseudo-Classes

Target elements in specific states or positions:

Example: Common Pseudo-Classes

/* Hover effect */
button:hover {
    background-color: #E8BB17;
    transform: translateY(-2px);
}

/* Focus state for form inputs */
input:focus {
    outline: 2px solid #0C2292;
    border-color: #0C2292;
}

/* First child element */
li:first-child {
    font-weight: bold;
    color: #0C2292;
}

/* Every odd element */
tr:nth-child(odd) {
    background-color: #f8f9fa;
}

Result (hover over these boxes):

Hover over me! (Box 1 - odd)
Hover over me! (Box 2 - even)
Hover over me! (Box 3 - odd)
Hover over me! (Box 4 - even)

Popular Pseudo-Classes

  • :hover - When mouse is over element
  • :focus - When element has focus
  • :active - When element is being clicked
  • :first-child - First child element
  • :last-child - Last child element
  • :nth-child(n) - Nth child element
  • :not(selector) - Elements that don't match selector

Combinators

Combine selectors to target elements based on their relationships:

1. Descendant Combinator (Space)

Example: Descendant Combinator

/* Selects ALL p elements inside div elements */
div p {
    color: blue;
    font-style: italic;
}

HTML:

<div>
    <p>This paragraph is inside a div</p>
    <section>
        <p>This paragraph is also inside the div (nested)</p>
    </section>
</div>
<p>This paragraph is NOT inside a div</p>

2. Child Combinator (>)

Example: Child Combinator

/* Selects ONLY direct children p elements of div */
div > p {
    background-color: rgba(232, 187, 23, 0.2);
    padding: 10px;
}

Result:

I'm a direct child of div (styled)

I'm a grandchild of div (not styled)

3. Adjacent Sibling Combinator (+)

Example: Adjacent Sibling

/* Selects p that immediately follows another p */
p + p {
    margin-top: 20px;
    background-color: rgba(12, 34, 146, 0.1);
}

Result:

First paragraph (no special styling)

Second paragraph (has special styling)

Third paragraph (has special styling)

CSS Specificity

When multiple CSS rules target the same element, specificity determines which style wins:

Specificity Hierarchy (Highest to Lowest):

  1. Inline styles (1000 points) - style="color: red;"
  2. IDs (100 points) - #myId
  3. Classes, attributes, pseudo-classes (10 points) - .myClass, [type="text"], :hover
  4. Elements and pseudo-elements (1 point) - div, p, ::before

Specificity Examples:

/* Specificity: 1 (element) */
p { color: black; }

/* Specificity: 10 (class) */
.highlight { color: blue; }

/* Specificity: 100 (ID) */
#special { color: red; }

/* Specificity: 111 (ID + class + element) */
div.highlight#special { color: green; }

Result (ID wins over class wins over element):

Element selector only (gray)

Class selector (gold background)

ID selector (blue background)

Multiple Selectors

Apply the same styles to multiple selectors using commas:

Example: Multiple Selectors

/* Apply same styles to multiple elements */
h1, h2, h3 {
    color: #0C2292;
    font-family: Arial, sans-serif;
}

/* Apply same styles to multiple classes */
.primary-button, .secondary-button, .danger-button {
    padding: 12px 24px;
    border-radius: 6px;
    border: none;
    cursor: pointer;
}

Practical Selector Examples

Real-World Selector Combinations:

/* Navigation menu styling */
nav ul li a {
    text-decoration: none;
    color: #0C2292;
    padding: 10px 15px;
}

nav ul li a:hover {
    background-color: #E8BB17;
    color: white;
}

/* Form styling */
form input[type="text"],
form input[type="email"],
form textarea {
    border: 2px solid #ddd;
    padding: 10px;
    border-radius: 4px;
}

form input[type="text"]:focus,
form input[type="email"]:focus,
form textarea:focus {
    border-color: #0C2292;
    outline: none;
}

/* Card styling */
.card:nth-child(odd) {
    background-color: #f8f9fa;
}

.card:last-child {
    margin-bottom: 0;
}

Best Practices

✅ Do:

  • Use classes for reusable styles
  • Use IDs for unique elements only
  • Keep specificity low when possible
  • Use meaningful class names (e.g., .primary-button not .blue-button)
  • Group related selectors together
  • Use combinators to target specific relationships

❌ Don't:

  • Overuse IDs - classes are usually better
  • Use !important unless absolutely necessary
  • Make selectors overly specific
  • Use inline styles for main styling
  • Use generic class names like .red or .big

Quick Reference

Selector Cheat Sheet:

element          /* h1, p, div */
.class           /* .highlight, .button */
#id              /* #header, #main */
*                /* All elements */
[attribute]      /* [type], [href] */
:pseudo-class    /* :hover, :focus */
element element  /* div p (descendant) */
element > element /* div > p (child) */
element + element /* p + p (adjacent) */
element, element  /* h1, h2 (multiple) */
